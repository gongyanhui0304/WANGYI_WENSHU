# 邮件项目分析原型

这是一个基于 Codex Skill 和本地静态文件的邮件项目分析验证原型。它不包含 Python 文件、数据库、后端、MCP 或独立前端。

当前语料来自用户放入的真实导出邮件：

- `emails/王慧/已发送`
- `emails/王慧/收件箱`

每封邮件是一个目录，正文通常为 `*_邮件提取.docx`，附件保留在对应 `attachments` 目录。原始邮件不被修改。

## 如何使用

在 Codex 中直接用自然语言提问，例如：

- 付款审批现在怎么样？
- 哪些专题风险最高？
- 王慧最近在处理什么？
- Bank Status 现在有什么问题？
- 摩洛哥订单的 SBLC 是谁在跟？
- 查看支持这个结论的原始邮件。
- 打开股东会审计专题的完整邮件线程。
- Invoice 61403337 后来有人回复了吗？

Codex 应使用 `skills/email-project-analysis/SKILL.md` 的流程：

1. 先读取 `index/projects.yaml` 或 `index/people.yaml`。
2. 通过专题名称、别名、客户、负责人或人员定位 `project_id`。
3. 优先读取项目摘要。
4. 只有问题涉及具体时间、邮件依据、完整沟通过程或摘要不足时，再读取时间线、线程摘要或原始邮件。
5. 不在每次提问时重新扫描全部邮件。
